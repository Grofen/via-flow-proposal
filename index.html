1. Executive Summary

Project Overview

ViaFlow is a comprehensive dispatch and quote management system designed specifically for transportation services specializing in disabled, senior, and specialized passenger transport. This modern, cloud-based solution will streamline operations, automate quote generation, and improve service delivery through real-time tracking and communication.

  • Enhance Customer Experience: Automated communications, transparent pricing, and easy booking
  • Improve Operations: Centralized dashboard, recurring trip management, and comprehensive reporting
  • Mobile Accessibility: Native mobile app for drivers with offline capabilities

4. Proposed Solution

We propose a comprehensive, cloud-based dispatch and quote management system built as a modern monorepo application. The solution consists of three main components working seamlessly together:

System Components

1. Admin Dashboard (Web Application)

A responsive web application built with Next.js 14+ providing:

  • Request Management: View, edit, and assign customer transportation requests
  • Quote Generation: Auto-calculated quotes with manual override capability
  • Driver Assignment: Assign vehicles and drivers to approved quotes
  • Real-time Tracking: Monitor ride status and driver locations
  • User Management: Role-based access control (Admin, Employee, Driver)
  • Reporting: Comprehensive analytics and reporting tools
  • Recurring Schedules: Manage recurring trips and auto-generate rides

2. Driver Mobile App (React Native)

A native mobile application for iOS and Android providing:

  • Assignment Notifications: Push notifications for new ride assignments
  • Ride Management: Accept/decline assignments and view ride details
  • Status Updates: Update ride status (En Route, Arrived, In Progress, Completed)
  • Navigation Integration: One-tap navigation to pickup/dropoff locations
  • Customer Communication: Click-to-call functionality
  • Offline Capability: View assigned rides without internet connection

3. Backend API (Node.js + tRPC)

A robust API server handling all business logic:

  • Pricing Engine: Intelligent quote calculation with distance, time, and surcharges
  • Authentication: Secure user authentication with better-auth
  • Real-time Updates: WebSocket connections for live status synchronization
  • Task Queue: Background job processing for emails, PDFs, and notifications
  • Audit Logging: Comprehensive logging of all system actions
  • API Integration: Connect with mapping, email, and SMS services

Key Innovation: Intelligent Pricing Engine

Our proprietary pricing engine automatically calculates accurate quotes based on:

  • Distance and estimated travel time from routing APIs
  • Vehicle type and capacity requirements
  • Time-based surcharges (night, weekend, holiday)
  • Location-based surcharges (airport, train station)
  • Waiting time and multiple stops
  • Custom pricing rules and discounts

Employees can review and manually adjust any quote before sending to customers.

2. Intelligent Quote Generation

Automated Pricing Engine

Our proprietary pricing engine calculates quotes automatically based on multiple factors:

Base Calculation

  • Distance: Calculated via Google Maps/Mapbox routing API
  • Travel Time: Estimated duration for the journey
  • Base Fee: Configurable flat rate per trip
  • Per-Kilometer Rate: Distance-based pricing
  • Per-Minute Rate: Time-based pricing component

Automatic Surcharges

  • Night Surcharge: 22:00 - 06:00 (configurable percentage)
  • Airport Surcharge: Automatic detection of airport locations
  • Waiting Time: Per-minute charges for delays
  • Multiple Stops: Additional fee per intermediate stop
  • Holiday Pricing: Special rates for holidays

Manual Override

  • Employees can adjust any quote component
  • Reason logging for manual changes
  • Price history and audit trail
  • Discount application with approval workflow

3. Quote Management Workflow

Draft Sent Accepted/Rejected

Complete lifecycle management:

Development Methodology

We employ an Agile development methodology with 2-week sprints, ensuring flexibility, iterative development, and regular feedback loops.

Sprint Structure

  • Sprint Planning: Define goals and tasks for the sprint
  • Daily Standups: 15-minute sync meetings
  • Sprint Review: Demonstrate completed work
  • Sprint Retrospective: Continuous improvement
  • Client Check-ins: Weekly progress updates

Monorepo Architecture

Why Monorepo?

Using Turborepo for monorepo management provides significant advantages:

  • Code Sharing: Share types, utilities, and components across all applications
  • Atomic Changes: Update all apps simultaneously with single commits
  • Consistent Tooling: One configuration for linting, testing, and building
  • Faster CI/CD: Smart caching and parallel builds
  • Type Safety: End-to-end type checking across the entire stack

API Design Principles

  • RESTful Principles: Clear resource naming and HTTP methods
  • Type Safety: tRPC procedures with Zod validation
  • Error Handling: Consistent error responses with proper status codes
  • Rate Limiting: Protection against abuse

Database Design

Prisma Schema Approach

  • Normalized Design: Minimal data duplication

  • Soft Deletes: Preserve data with deletedAt timestamps
  • Audit Fields: createdAt, updatedAt, createdBy on all tables
  • Enum Types: Type-safe status and role enums
  • JSON Fields: Flexible metadata storage where appropriate

Performance Optimization

  • Server-Side Rendering: Fast initial page loads with Next.js
  • Code Splitting: Lazy load components for smaller bundles
  • Image Optimization: Next.js automatic image optimization
  • Database Indexing: Strategic indexes on frequently queried fields
  • Query Optimization: Efficient Prisma queries with select and include
  • Caching Strategy: Redis for session data and frequently accessed resources

Appendix C: FAQ

General Questions

Q: Why a monorepo architecture?

A: A monorepo allows us to share code between the dashboard, mobile app, and API, ensuring consistency and reducing duplication. It also enables atomic changes across all applications and simplifies dependency management.

Q: Can we add features after the MVP launch?

A: Absolutely! The system is designed to be extensible. We can add new features through additional development sprints on a time-and-materials basis or through a monthly retainer.

Q: What if we want to self-host instead of using managed services?

A: We can adapt the deployment strategy for self-hosting. This may require additional infrastructure setup and ongoing maintenance, but the application is fully compatible with self-hosted environments.

Technical Questions

Q: Why tRPC instead of REST or GraphQL?

A: tRPC provides end-to-end type safety without code generation, reducing bugs and improving developer experience. For your use case, it offers the perfect balance of type safety and simplicity. We can switch to REST if preferred.

Q: Can the system handle multiple languages?

A: Yes, internationalization (i18n) can be added. The MVP will be in English/German (specify), with additional languages available as an enhancement.

Q: What happens if a third-party service (like Google Maps) goes down?

A: The system includes error handling and fallback mechanisms. For critical services, we implement retry logic and graceful degradation. Quotes can still be created manually if APIs are unavailable.

Q: Is the system mobile-responsive?

A: Yes, the dashboard is fully responsive and works on tablets and mobile devices. However, for the best driver experience, we recommend the dedicated React Native mobile app.

Security Questions

Q: How is customer data protected?

A: All data is encrypted in transit (TLS 1.3) and at rest. Access is controlled through role-based permissions. We follow GDPR requirements and industry best practices for data protection.

Q: Can we perform our own security audit?

A: Yes, we welcome third-party security audits. We'll provide full access to the codebase and documentation to facilitate the audit process.

Q: What authentication methods are supported?

A: The MVP includes password-based authentication with optional 2FA. We can add magic links, social login, or single sign-on (SSO) as enhancements.

Business Questions

Q: Can we customize the pricing rules?

A: Yes, pricing rules are fully configurable through the admin interface. You can set base fees, per-kilometer rates, time-based pricing, and various surcharges without code changes.

Q: How do recurring trips work?

A: Set up a recurring schedule once (e.g., "every Monday and Wednesday at 8 AM for 3 months"), and the system automatically generates individual rides. Each ride can still be modified or cancelled independently.

Q: Can customers access the system directly?

A: The MVP focuses on the request intake form (public) and employee/driver interfaces. A customer portal for tracking bookings and viewing history is available as Phase 3.

Q: What reports are included?

A: The MVP includes basic reports: request volume, quote conversion, driver performance, and revenue by service type. Advanced analytics and custom reports can be added post-MVP.

Support Questions

Q: What training is provided?

A: We provide up to 4 hours of training sessions for employees and drivers, plus comprehensive documentation. Additional training is available at €150/hour.

Q: What happens after the 30-day warranty period?

A: You can continue with monthly support (€500/month) for ongoing maintenance and minor enhancements, or engage us on a project basis for larger features.

Q: Can you help with future integrations?

A: Yes, we can integrate with accounting software, payment gateways, or other systems. Integration projects are quoted separately based on complexity.
